# 厦门室内高尔夫+射击体验店微信小程序配置规格书

## 一、概述

### 1.1 项目背景

厦门室内高尔夫+射击体验店采用「半无人」运营模式，基于纳客管理系统（久久客版）实现微信小程序在线预约、智能门锁控制、自动计时计费、会员储值管理等核心功能。本规格书旨在明确纳客系统在本项目中的配置需求，确保系统功能与实体店运营流程无缝对接。

### 1.2 小程序定位

- **产品名称**：[待定] 高尔夫射击体验馆微信小程序
- **核心功能**：在线预约、移动支付、会员卡管理、智能设备控制
- **技术基础**：纳客管理系统久久客版（买断版，¥10,000-15,000，免年费）
- **配置性质**：基于纳客现有功能进行客制化参数配置，非二次开发

### 1.3 目标用户

- **散客**：临时体验用户，通过小程序预约或到店扫码即时预约
- **会员**：办理次卡/月卡用户，享受折扣优惠和积分奖励
- **团建客户**：企业/团队包场需求，提前预约多个单元
- **临时到店客户（Walk-in）**：无预约用户，到店扫码查看当前可用时段并即时预约

## 二、预约模块配置

### 2.1 时段设置

| 参数 | 配置值 |
|------|--------|
| 时段粒度 | 30 分钟/时段 |
| 营业时间 | 08:00 - 24:00（每日） |
| 可预约时段 | 08:00, 08:30, 09:00, ..., 23:00, 23:30（共 32 个时段） |
| 提前预约 | 至少提前 15 分钟预约（防止系统准备不及） |
| 最长预约 | 单次预约最多 4 小时（8 个连续时段） |

### 2.2 单元选择

- **展示内容**：
  - 单元编号（如「1 号单元」「2 号单元」）
  - 单元规格：22-48㎡，配备 DTG3 高尔夫模拟器 + ShootSim 射击模拟器
  - 实时状态：
    - 🟢 可用（该时段无预约）
    - 🔴 已预约（显示预约模式：高尔夫/射击/套餐）
    - 🟡 使用中（当前正在使用）

- **选择逻辑**：
  - 用户选择时段后，系统显示该时段所有单元的可用状态
  - 仅可选择「🟢 可用」状态的单元
  - 支持多单元同时预约（团建场景）

### 2.3 模式选择

用户确认单元和时段后，选择体验模式：

| 模式 | 时长 | 说明 |
|------|------|------|
| 高尔夫 | 1 小时起（可延长） | 使用 DTG3 保护器，启动 DTG3 软件 |
| 射击体验 | 30 分钟起（可延长） | 使用 ShootSim 保护器，启动 ShootSim 软件 |
| 高尔夫+射击套餐 | 1.5 小时（固定） | 0.75 小时高尔夫 + 0.75 小时射击，中途自动切换模式提醒 |

### 2.4 关键技术说明（保护器使用原则）

📌 **ShootSim 与 DTG3 保护器的正确使用方式**

**技术原理**：
- ShootSim 保护器（dongle）仅在初始设置和校准时需要使用
- 校准完成后，ShootSim 可以在无保护器的情况下正常运行
- DTG3 保护器在整个高尔夫体验过程中保持插入
- **关键结论**：同一单元可以在同一时段容纳高尔夫与射击两种预约

**实际操作流程**：
1. **初次使用 ShootSim**（仅需一次）：插入 ShootSim 保护器 → 启动 Shootsim.exe 进行校准 → 校准完成
2. **后续 ShootSim 使用**：可拔除保护器 → 直接启动 mosemu.exe → ShootSim 运行无需保护器
3. **高尔夫使用**：DTG3 保护器保持插入 → 启动 DTG3 软件 → 正常体验
4. **高尔夫+射击套餐**：无需切换保护器 → 中途自动切换软件界面提醒 → 继续体验

**预约配置说明**：
- 同一单元同一时段可被预约为：高尔夫 OR 射击 OR 高尔夫+射击套餐
- 不同用户可在同一单元同一时段预约不同模式（系统需支持灵活的预约选项）
- 用户自助选择体验模式，系统自动处理计费和时间管理

**纳客系统配置要求**：
- 支持同一单元同一时段的多模式预约灵活性
- 预约成功后显示该单元该时段的预约状态（高尔夫/射击/套餐）
- 支持"高尔夫+射击"组合套餐的中途自动切换模式提醒功能

### 2.5 临时客户（Walk-in）处理

**场景**：用户未提前预约，直接到店扫码

**配置流程**：
1. 到店后扫描门口二维码 → 进入小程序
2. 小程序自动定位当前时段（如当前时间 14:23，显示 14:30、15:00、15:30... 等后续可用时段）
3. 显示各单元实时状态（🟢 可用 / 🔴 已预约 / 🟡 使用中）
4. 用户选择可用单元 + 模式 → 完成支付 → 立即获得开锁权限

**配置要点**：
- 启用「即时预约」功能，允许当前时段 + 未来时段预约
- Walk-in 用户与预约用户享受相同计费规则
- 如当前所有单元占满，显示「当前无可用单元，请稍后再试或预约未来时段」

### 2.6 超时处理

| 阶段 | 配置动作 |
|------|----------|
| 预约时间结束前 5 分钟 | 小程序推送提醒：「您的预约即将结束，如需延时请点击续费」 |
| 用户选择续费 | 弹出续费选项：+30 分钟 / +1 小时，按实时价格计费（工作日/周末自动识别） |
| 用户确认支付 | 自动延长使用时间，设备继续开放 |
| 预约时间结束且未续费 | 自动结束计时，5 分钟后执行离店流程（关灯+关冷气+锁门） |

**纳客配置需求**：
- 设置「到时提醒」功能，提前 5 分钟推送微信模板消息
- 启用「在线续费」功能，支持微信支付实时扣款
- 超时未离店处理：结束后 5 分钟宽限期，之后自动触发智能设备关闭流程

## 四、付款配置

### 4.1 支付方式

| 支付方式 | 状态 | 说明 |
|----------|------|------|
| 微信支付（WeChat Pay） | ✅ 必须启用 | 纳客系统默认支持，配置商户号即可 |
| 支付宝 | ❓ 待确认 | 需与纳客销售确认是否支持（久久客版） |
| 储值卡扣款 | ✅ 必须启用 | 会员储值余额直接扣款 |
| 次卡/月卡扣次 | ✅ 必须启用 | 10 次卡扣次，月卡免费使用 |

### 4.2 支付流程

```
用户选择模式 + 时段 → 系统计算费用 → 选择支付方式 
   ↓
[微信支付] → 调起微信支付 → 支付成功 → 预约确认 → 发送预约凭证
[储值卡] → 检查余额 → 余额充足扣款 → 预约确认 → 发送预约凭证
[次卡] → 检查剩余次数 → 扣除 1 次 → 预约确认 → 发送预约凭证
[月卡] → 检查有效期 → 月卡有效 → 预约确认 → 发送预约凭证
```

### 4.3 支付配置要点

- **微信支付**：需提供商户号（MCHID）、API 密钥（API Key）、商户证书
- **退款规则**：
  - 预约时间前 2 小时以上取消 → 全额退款
  - 预约时间前 2 小时内取消 → 不予退款
  - 次卡/月卡用户取消预约 → 退还扣除的次数/不影响月卡权益
- **发票配置**：（待确认）是否支持电子发票自动开具功能

## 六、智能设备控制配置

### 6.1 门锁控制

| 触发条件 | 动作 | 配置要点 |
|----------|------|----------|
| 用户到店扫码 + 预约时间已到 | 自动开锁（对应单元门） | 门锁与单元绑定，仅开放已预约单元 |
| 用户离店（主动结束或超时） | 自动上锁（延迟 5 分钟） | 给用户留出离开时间，防止误锁 |
| 预约时间未到提前扫码 | 不开锁，提示「预约时间未到，请稍候」 | 防止提前占用资源 |

**纳客配置需求**：
- 确认纳客系统与智能门锁品牌兼容性（如 TTLock、果加、鹿客等）
- 配置门锁设备 ID 与单元编号的映射关系

### 6.2 灯光控制

| 触发条件 | 动作 | 配置要点 |
|----------|------|----------|
| 用户扫码开锁成功 | 自动开灯（对应单元） | 联动智能开关，照明单元内所有灯具 |
| 预约时间结束 + 用户离店 | 自动关灯（延迟 5 分钟） | 与门锁同步执行，确保安全离店 |

**纳客配置需求**：
- 确认纳客系统与智能灯光品牌兼容性（如 Yeelight、欧瑞博、绿米等）
- 配置智能开关设备 ID 与单元编号的映射关系

### 6.3 冷气（空调）控制

| 触发条件 | 动作 | 配置要点 |
|----------|------|----------|
| 用户扫码开锁成功 | 自动开启空调（对应单元，制冷模式，24°C） | 夏季制冷，冬季制热（根据季节预设模式） |
| 预约时间结束 + 用户离店 | 自动关闭空调（延迟 5 分钟） | 节能环保，降低运营成本 |

**纳客配置需求**：
- 确认纳客系统与智能空调品牌兼容性（如 Broadlink、米家、天猫精灵等红外/WiFi 控制）
- 配置空调设备 ID 与单元编号的映射关系
- 设置季节性温度预设（夏季 24°C 制冷，冬季 22°C 制热）

### 6.4 设备控制时序

```
用户到店扫码（预约时间已到）
   ↓
[1] 验证预约有效 → 开锁授权（3 秒内执行）
   ↓
[2] 门锁打开 → 触发灯光开启（5 秒内执行）
   ↓
[3] 灯光开启 → 触发空调开启（10 秒内执行）
   ↓
用户进入单元 → 按墙面指示插入保护器 → 启动软件 → 开始体验

----------------------------------------

预约时间结束（或用户主动结束）
   ↓
[1] 小程序推送「请离店」提醒
   ↓
[2] 5 分钟宽限期后 → 自动关闭空调
   ↓
[3] 2 分钟后 → 自动关闭灯光
   ↓
[4] 1 分钟后 → 自动上锁
```

**纳客配置需求**：
- 设置设备控制时序，确保开锁 → 开灯 → 开空调的顺序执行
- 设置离店延迟关闭时间（空调 5 分钟 → 灯光 7 分钟 → 门锁 8 分钟）
